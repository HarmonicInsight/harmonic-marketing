# SFAD パッケージ 販売・マーケティング戦略

> 作成日: 2026-04-02
> 目的: SFADメソドロジーをパッケージ商品化し、高単価B2B教育商品として販売する戦略

---

## エグゼクティブサマリー

SFADは「Screen First AI Development」の略で、**AIを活用したWebアプリケーション開発メソドロジー**。
従来のSI開発（要件定義→設計→実装→テスト）の非効率を解消し、
**プロトタイプ駆動 × AI × 10ステップ**で、非エンジニアでも業務アプリを構築できる方法論。

### 既存製品との位置づけ

```
HARMONIC 製品ポートフォリオ
├── ツール販売（¥0〜¥49,800）
│   ├── IAOF, INSS, IOSH, IOSD ... → デスクトップツール
│   └── HARMONIC tools → 個人向け小ツール
│
├── SFAD パッケージ（¥298,000〜¥498,000）★本戦略
│   └── メソドロジー + 教材 + コード + 動画 → 開発手法パッケージ
│
└── 導入支援サービス（案件ベース）
    └── キックオフ + 伴走型支援 + カスタム開発
```

**SFADは「ツール」ではなく「方法論の体系」を売る商品。**
既存のツール販売（単価¥49,800以下）とは全く異なるセグメント。

## 1. SIPOとSFADの関係整理

SIPOとSFADは**補完関係**にある。混同を避け、明確に使い分ける。

| | SIPO | SFAD |
|---|---|---|
| **What** | 業務分析フレームワーク | 開発メソドロジー |
| **焦点** | 「何を作るべきか」を特定する | 「どう作るか」を体系化する |
| **対象** | 経営企画・DX推進・コンサルタント | PM・企画職・DX推進・内製化チーム |
| **アウトプット** | 業務の情報源（Source）の可視化 | 動くWebアプリケーション |
| **価格帯** | セミナー ¥15,000〜¥30,000 | パッケージ ¥298,000〜¥498,000 |

```
顧客の課題解決ジャーニー

  [SIPO]                          [SFAD]
  業務を分析・設計する              設計を元にアプリを作る
  ────────────────────────────────────────────────────
  Source特定 → 業務フロー設計  →  Step1-10で実装 → 本番リリース
                                  ↑
                            ここがSFADの領域
```

**メッセージの使い分け:**
- SIPO:「業務の情報源を理解すれば、DXの80%は終わる」
- SFAD:「理解した業務を、AIと一緒に10ステップでアプリにする」

## 2. ターゲット顧客

### プライマリ: 中堅企業のDX推進部門

| 属性 | 詳細 |
|---|---|
| 企業規模 | 従業員100〜1,000名 |
| 業種 | 建設、製造、物流、不動産 |
| 部門 | DX推進室、情報システム部、経営企画 |
| 課題 | SI依存のコスト・時間・属人化 |
| 予算 | 年間IT予算 500万〜5,000万 |
| 意思決定者 | DX推進責任者、CTO、情報システム部長 |

### セカンダリ: コンサルタント・SIer

| 属性 | 詳細 |
|---|---|
| 対象 | 独立系コンサルタント、中小SIer |
| 動機 | 自社のサービスメニューに追加したい |
| 価値 | SFADを使ってクライアントに提供 → 差別化 |
| 展開 | パートナーライセンス（将来） |

### ペインポイント（課題）

1. **SIer依存**: 小さな修正でも見積→発注→1ヶ月待ち
2. **コスト**: 業務アプリ1本に500万〜3,000万
3. **属人化**: 作った人しかわからない。引き継ぎ地獄
4. **乖離**: 要件定義書と実際のアプリが全然違うものになる
5. **スピード**: 市場環境の変化にシステムが追いつかない

| リスク | 影響 | 対策 |
|---|---|---|
| 「メソドロジーにお金を払う」文化がない | 受注ゼロ | 体験ワークショップで価値を体感させる。ROI試算を提案書に含める |
| Claude Code の API 料金が顧客に嫌がられる | 購入後の不満 | API料金の目安を教材に明記。月額¥5,000-¥20,000程度と提示 |
| 顧客のITリテラシーが低すぎて自走できない | 解約・クレーム | Enterprise プランで伴走支援をセット販売。Standard は自走力ある顧客に限定 |
| 競合（ノーコードツール）との比較 | 価格負け | 「ノーコードの限界」を明確に訴求。カスタマイズ性・拡張性で差別化 |
| 制作リソース不足（一人で全制作） | スケジュール遅延 | Phase 2 の教材制作はAI（Claude）を最大活用。完璧より完了を優先 |

## 10. 既存マーケティング戦略との統合

### コンテンツカレンダーへの組み込み

既存の12週間カレンダー（execution-calendar-12weeks.md）に、SFAD関連コンテンツを週1本ペースで追加。
ただし既存のINMV/IAOF/SIPOコンテンツと競合しないよう、曜日を分ける。

```
月: Twitter（既存 or SFAD）
火: Shorts（既存）
水: note（既存 or SFAD）
木: ★SFAD専用（YouTube本編 or note記事）← 新設
金: YouTube本編（既存）
土: Shorts（SFAD）
```

### CTA設計

SFAD関連コンテンツのCTAは既存の3段階CTAに準拠:

```
1. 「SFADについてもっと知りたい方」
   → LP / 資料DL（概要欄）

2. 「体験してみたい方」
   → 無料ウェビナー / 体験ワークショップ（概要欄）

3. 「自社で導入したい方」
   → 問い合わせフォーム（概要欄）
```

### ブランド

SFADは **HARMONIC insight（法人向けブランド）** の傘下に位置づける。
HARMONIC tools（個人向け）には含めない。

```
HARMONIC insight
├── Insight Business Suite（ツール群）
└── SFAD（開発メソドロジー）★ここ
```
