# コンテンツ → 製品購入・セミナー参加 導線設計

> 作成日: 2026-03-21
> 目的: コンテンツを「自慢」ではなく「課題解決」に軸を置き、製品購入・セミナー参加に自然に繋げる

---

## 根本方針: コンテンツの役割を再定義する

```
❌ 従来の発想
  「私はすごい」→「すごいでしょ？」→ 視聴者「すごいね」→ 終わり

✅ あるべき発想
  「あなたのこの課題」→「こうやって解決できます」→「もっと深くやりたいなら」
    → 製品を買う / セミナーに来る / コンサルを依頼する
```

**キャリアの数字（100社以上、4.5億、140リポ）は「信頼性の裏付け」として使う。メインコンテンツにしない。**

## 2. コンテンツの3層構造

### 層1: 課題提起（無料・広くリーチ）

**目的**: 視聴者に「それ、うちもだ」と思わせる
**プラットフォーム**: YouTube Shorts, Twitter

| コンテンツ例 | 訴求する課題 | 導線先 |
|---|---|---|
| 「社内研修の動画、まだ外注してますか？」 | 研修動画制作の手間とコスト | INMV |
| 「会議の後、議事録書くのに30分かかってませんか？」 | 議事録作成の非効率 | IAOF |
| 「Excelの集計、毎月同じ作業してませんか？」 | 定型業務の自動化 | STORESツール |
| 「建設現場の情報、ちゃんと経営に届いてますか？」 | 情報伝達の断絶 | SIPO セミナー |
| 「提案書、パワポだけで伝わると思いますか？」 | 合意形成の非効率 | プロトタイプ駆動セミナー |

### 層2: 解決方法の実演（無料・信頼構築）

**目的**: 「こうやれば解決できる」を具体的に見せる
**プラットフォーム**: YouTube本編, note無料記事

| コンテンツ例 | 実演内容 | 製品接点 |
|---|---|---|
| 「PPTから研修動画を5分で作る」 | INMVの実演（画面収録） | 概要欄にINMV |
| 「AIで会議議事録を自動化する方法」 | IAOFの実演 | 概要欄にIAOF |
| 「SIPOフレームワークで業務を再設計する」 | フレームワーク解説+事例 | セミナー告知 |
| 「プロトタイプ駆動型で提案する方法」 | Claude Codeでプロトタイプ作成 | セミナー告知 |
| 「建設業の基幹システム、カスタマイズが多発する理由」 | SIPO 4層モデルで分析 | コンサル問い合わせ |

**ポイント: ここで「瀬田の実績」は信頼性の裏付けとして触れる（メインにしない）**
例: 「100社以上のシステム導入・コンサルティングをやってきた経験から言うと、こういう業務設計の問題は何十回も見てきました。その中で……」

**目的**: 「もっとやりたい」人を製品購入・セミナーに転換
**プラットフォーム**: STORES, セミナー, note有料

| コンテンツ → 転換先 | 価格 | 提供内容 |

| YouTube実演 → INMV購入 | 法人見積 | Training Studioライセンス |
| YouTube実演 → STORES教材 | ¥3,000-¥5,000 | 「PPT→動画化 完全マニュアル」 |
| SIPO解説 → セミナー | ¥10,000-¥30,000 | 「SIPOフレームワーク実践ワークショップ」 |
| プロトタイプ駆動 → セミナー | ¥15,000-¥30,000 | 「AI×プロトタイプ駆動型コンサルティング実践講座」 |

## 4. コンテンツ別CTA設計

各コンテンツの末尾に何を置くかを統一する。

### YouTube本編のCTA構造

```
【動画の最後 — 3段階CTA】

1. 「今日の内容を自分でやってみたい方」
   → STORES / HARMONIC insight のリンク（概要欄）

2. 「体系的に学びたい方」
   → セミナーの申込リンク（概要欄）

3. 「自社でやりたい方」
   → お問い合わせフォーム（概要欄）

※「チャンネル登録」は最後ではなく、動画冒頭5分以内に軽く触れる
```

### note記事のCTA構造

```
【記事の末尾 — 2段階CTA】

1. 「もっと深く学びたい方」
   → 有料記事 or セミナーリンク

2. 「実際のツールを試したい方」
   → STORES / HARMONIC insight リンク

※ 有料記事の場合は「次の記事」へのリンクで連続購入を促す
```

### YouTube ShortsのCTA構造

```
【最後3秒】
「解決方法は本編で」→ 本編動画リンク

※ Shortsは「課題提起」に徹する。売り込みは絶対にしない。
```

## 6. YouTube VID-022 の構成修正方針

**変更の核心**: 自己紹介と実績の自慢を減らし、「視聴者の課題解決」を前面に出す

### Before（現在の構成）
1. タイトル
2. **自己紹介（実績列挙）** ← 長い
3. 28年前と今の違い
4. 上流経験者が最強な5つの理由 ← 自分の話
5. 具体例 — 私が作ったもの ← 自分の話
6. プロトタイプ駆動型の実演
7. 当たり前が武器になる
8. 反論への回答
9. キャリア戦略
10. まとめ

### After（修正後の構成）
1. **課題提起**: 「提案書だけで伝わらない経験、ありませんか？」← 視聴者の課題から始める
2. **自己紹介（30秒で）**: 実績は信頼性の裏付けとして最小限
3. **解決方法**: プロトタイプ駆動型コンサルティングの紹介
4. **実演**: 業務理解→提案→プロトタイプ→フィードバック
5. **なぜこれができるのか**: 上流経験がAIで解放される（ここで経験の話）
6. **あなたもできる**: 視聴者への動機付け
7. **反論への回答**
8. **まとめ + 3段階CTA**: 製品 / セミナー / コンサル

## 7. 経歴の使い方ガイドライン

### ✅ OK: 信頼性の裏付けとして使う
- 「100社以上にシステム導入してきた経験から言うと……」
- 「建設DXの現場で実際に見てきた問題は……」
- 「UiPathの導入支援を30社以上やった中で学んだのは……」

### ❌ NG: メインコンテンツにする
- 「私は電通総研で……PwCで……FPTで……（3分間の経歴紹介）」
- 「140リポジトリ作りました！40冊出版しました！」
- 「売上を8千万から4.5億に伸ばしました」

### ルール
- 自己紹介は**30秒以内**
- 経歴は**課題解決の文脈で**のみ言及
- 数字は**根拠として**使う（「100社以上で見てきた結果」）、**自慢として**使わない
