## なぜ建設業で管理会計が機能しないのか

ほとんどの建設会社で「管理会計」が回っていない。
理由はシンプルで、**財務会計（税務申告のための会計）しかやっていないから**である。

| 財務会計 | 管理会計 |
|---------|---------|
| 法律で義務づけられている | 任意（でも経営には必須） |
| 過去の結果を記録する | 未来の意思決定に使う |
| 勘定科目・月次で集計 | 工事別・工区別・業者別で集計 |
| 年度末に利益がわかる | 今月時点で利益がわかる |
| 税理士が見る | 経営者・現場所長が見る |

建設業では、工事が終わってから「あの工事は赤字だった」と判明するケースが非常に多い。
**これは管理会計が存在しないことの直接的な結果**である。

## 2. 原価管理の4要素

### 2.1 原価構成

```
工事原価
  ├── 材料費   ── 資材の購入費用
  ├── 外注費   ── 協力業者への支払い（建設業で最大の原価要素、通常50-70%）
  ├── 労務費   ── 自社作業員の人件費
  └── 経費     ── 仮設、重機リース、運搬、保険等
```

### 2.2 予算と実績の対比テンプレート

| 原価要素 | 予算 | 当月実績 | 累計実績 | 残予算 | 消化率 | 出来高率 | 判定 |
|---------|------|---------|---------|--------|-------|---------|------|
| 材料費 | 20,000 | 2,000 | 10,000 | 10,000 | 50% | 45% | ⚠ 超過傾向 |
| 外注費 | 50,000 | 5,000 | 25,000 | 25,000 | 50% | 45% | ⚠ 超過傾向 |
| 労務費 | 10,000 | 800 | 4,000 | 6,000 | 40% | 45% | ✓ 正常 |
| 経費 | 5,000 | 300 | 1,500 | 3,500 | 30% | 45% | ✓ 正常 |
| **合計** | **85,000** | **8,100** | **40,500** | **44,500** | **47.6%** | **45%** | **⚠** |

**判定ロジック**: 原価消化率が出来高率を5%以上超過 → ⚠アラート

## 3. 資金繰り管理テンプレート（工事別）

### 3.1 入出金予定表

| 月 | 入金予定 | 出金予定 | 月次CF | 累計CF |
|----|---------|---------|--------|--------|
| 4月 | 0 | ▲8,000 | ▲8,000 | ▲8,000 |
| 5月 | 20,000 | ▲8,500 | +11,500 | +3,500 |
| 6月 | 0 | ▲9,000 | ▲9,000 | ▲5,500 |
| 7月 | 25,000 | ▲8,500 | +16,500 | +11,000 |
| ... | | | | |
| 3月（完成） | 30,000 | ▲5,000 | +25,000 | +20,000 |

**ポイント**: 入金と出金のタイミング差で「利益は出ているのに資金が足りない」が起こる。工事別CFを把握していないと資金ショートの予兆が見えない。

### 3.2 全社資金繰り集約

```
全社CF = Σ(各工事CF) + 一般管理費CF + 借入返済CF + 設備投資CF

工事A: +3,500万（当月入金あり）
工事B: ▲2,000万（出来高先行・入金来月）
工事C: ▲500万（完成引渡前）
一般管理費: ▲1,500万
借入返済: ▲500万
─────────────
全社当月CF: ▲1,000万
```

## 4. 消し込みと管理会計の接続

### 4.1 消し込みが管理会計に与える影響

消し込みは単なる「事務処理」ではない。**管理会計の精度を決定する重要な業務**である。

| 消し込みの状態 | 管理会計への影響 |
|-------------|---------------|
| 消込が正確・タイムリー | 売掛金・買掛金残高が正確 → 工事別損益が信頼できる |
| 消込が遅延 | 入金済みなのに未収入金に残る → 回収状況が読めない |
| 消込が不正確 | 別工事の入金を誤消込 → 工事別損益がずれる |
| 消込ができていない | 未消込残高が膨らむ → 実態がわからなくなる |

### 4.2 消し込みから管理会計へのデータフロー

```
[消込完了]
    │
    ├──→ 売掛金残高更新 ──→ 工事別未収入金レポート
    │                         → 回収遅延アラート
    │
    ├──→ 買掛金残高更新 ──→ 工事別未払金レポート
    │                         → 支払予定管理
    │
    ├──→ 入金実績記録 ──→ 工事別CF実績
    │                       → 資金繰り予測の精度向上
    │
    └──→ 原価確定 ──→ 工事別損益更新
                        → 着地見込み更新
                        → 予算超過アラート
```

## 5. KPIダッシュボード設計

### 経営者向け（全社俯瞰）

| KPI | 計算式 | 目標値 | アラート閾値 |
|-----|--------|--------|------------|
| 全社粗利率 | 粗利合計 / 売上合計 | 20%以上 | 15%を下回ったら |
| 受注残高 | 契約済み未完成工事の残高合計 | - | 6ヶ月分の売上を下回ったら |
| 未消込残高 | 消込未済の売掛金+買掛金 | 月商の10%以内 | 20%を超えたら |
| 回収遅延率 | 支払期日超過の売掛金 / 売掛金合計 | 5%以内 | 10%を超えたら |
| 赤字工事件数 | 着地見込粗利がマイナスの工事数 | 0件 | 1件でも |

### 現場所長向け（担当工事）

| KPI | 計算式 | 見るべきタイミング |
|-----|--------|--------------------|
| 出来高率 vs 原価消化率 | 両方を対比 | 毎週 |
| 予算残 | 予算 - 実績累計 | 毎週 |
| 着地粗利見込 | 売上 - 原価見込 | 毎月 |
| 外注費消化率 | 外注費累計 / 外注費予算 | 毎月 |
| 未請求出来高 | 出来高金額 - 請求済金額 | 毎月 |

### 経理担当向け（消込・月次）

| KPI | 計算式 | 見るべきタイミング |
|-----|--------|--------------------|
| 自動消込率 | 自動消込件数 / 全消込件数 | 毎月（改善指標） |
| 未消込日数 | 入金から消込完了までの平均日数 | 毎月 |
| 月次クロージング日数 | 月末から決算確定までの日数 | 毎月 |
| 消込差異件数 | 差額調整が必要だった件数 | 毎月 |

## 6. 実装の優先順位

### Tier 1: これがないと始まらない（Month 1-2）

1. **工事番号の統一** — 全システムで同じ工事番号を使えるようにする
2. **工事カードの運用開始** — 1工事1カードで契約・予算・実績を一元管理
3. **月次の工事別損益レポート** — Excelでもいいので毎月出す

### Tier 2: これで管理会計が回り始める（Month 3-4）

4. **予算vs実績の対比** — 原価4要素ごとに予算消化率を追う
5. **消し込みルールの整備** — 例外パターンの分類と対応ルール策定
6. **入出金予定表** — 工事別の資金繰りを可視化

### Tier 3: これでDXの効果が出る（Month 5-6）

7. **自動消込の導入** — ルールに基づく消込自動化
8. **KPIダッシュボード** — リアルタイムに工事別損益を見られる
9. **アラート機能** — 予算超過・回収遅延の自動通知

## 7. Excel → システム移行の判断基準

| 規模 | 年間完工高 | 同時進行工事数 | 推奨 |
|------|-----------|--------------|------|
| 小規模 | 〜5億 | 〜10件 | Excel管理で十分。テンプレートを整備して運用 |
| 中規模 | 5〜30億 | 10〜50件 | Excel限界。工事管理システム＋消込ツール導入 |
| 大規模 | 30億〜 | 50件〜 | ERP or 建設業特化システム必須 |

**どの規模でも共通して必要なこと**:
- 工事番号体系の統一
- 月次の工事別損益把握
- 消し込みルールの明文化

これらはシステムの有無に関係なく、**業務設計の問題**である。
