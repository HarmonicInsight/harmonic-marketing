# Are You Designing Processes With IPO Alone? — Add One Element and Reproducibility Changes

You draw a workflow. You define Input, Process, Output.
It looks clean. It looks correct.

But is your Input actually correct?

Is it a number someone heard on a phone call? The latest version of a quote? A customer-approved figure?

**You can define Input items — but you can't guarantee Input accuracy.**
This is the core problem with IPO-only process design.

---

## Why IPO Alone Falls Short

The IPO model is everywhere:

- Input: Enter contract information
- Process: Approve
- Output: Issue purchase order

Looks fine — until you ask one question:

**"Where did that contract information come from, and who provided it?"**

Sales has one number in their spreadsheet. The ERP has another. The meeting notes say something different. This happens because the **Source** — the origin of the information — was never defined.

---

## What Is the SIPO Framework?

SIPO (Source → Input → Process → Output) adds one element before the traditional IPO model.

| Element | Question | Role |
|---------|----------|------|
| **S** – Source | Where does it come from? | Defines the authoritative origin |
| **I** – Input | What goes in? | A copy derived from Source |
| **P** – Process | How is it processed? | Follow defined procedures |
| **O** – Output | What comes out? | The deliverable |

Source is not just "raw data."
It is **the single point of truth — the decision of which fact is authoritative.**

---

## Source Means Both "Which Document" and "Which Definition"

Identifying the right document is not enough. Source only works when you also confirm **which definition** applies.

### Revenue
Completed contract revenue? Progress-based revenue? Billed amount?
If the definition differs, the same word "revenue" **produces completely different numbers.**

### Order Backlog
Gross orders? Net orders? Including carryover?
Ambiguous definitions cause **different numbers for different report recipients.**

### Unit Price
Price per m²? Lump sum? Labor rate?
Without aligned definitions, **you can't even compare quotes.**

**Source functions only when both the document and its definition are confirmed.**
This alone eliminates the "same word, different numbers" problem.

---

## Case Study 1: Meeting Minutes

### IPO Only
1. **Input**: A's memory of the meeting
2. **Process**: Summarize (freestyle)
3. **Output**: Minutes (quality varies by person)

### With SIPO
1. **Source**: Audio recording (the original)
2. **Input**: Transcript
3. **Process**: Summarize and extract
4. **Output**: Minutes (consistent quality regardless of author)

Define Source, and **anyone produces the same result.**

---

## Case Study 2: Purchase Order Process

| Element | Content |
|---------|---------|
| **Source** | Customer-approved contract |
| **Input** | Contract amount, order quantity |
| **Process** | Spec verification, budget check |
| **Output** | Approved purchase order |

When Sales' spreadsheet and the ERP disagree — if Source is defined as "the contract," **there is no debate.**

Staff changes? The new person pulls the same Input from the same Source. 100% reproducible.

---

## SIPO Is a Chain

SIPO does not end with a single process. **It chains.**

The Output of one process becomes the Source of the next.

For example, the "approved purchase order" from the ordering process becomes the **Source** for the downstream project management process.

> Output₁ → Source₂ of the next process

Design one Source correctly, and the Output it produces is also correct — **stabilizing every downstream process.**

---

## What Changes When You Add "S"

| Dimension | IPO Only | SIPO |
|-----------|----------|------|
| Information reliability | Depends on who entered it | Source guarantees correctness |
| Staff turnover | Know-how leaves with the person | Source and procedures remain |
| AI delegation | Unclear what to automate | Source-verified steps can be automated |

**One letter changes reliability, reproducibility, and accountability.**

---

## Three Steps to Start SIPO Analysis

### Step 1: Identify the Source
What is the "original" for this process? Define one authoritative information source.

### Step 2: Decompose into SIPO
Source → Input → Process → Output. Write out each step.

### Step 3: Define the Automation Boundary
- Source-verified → delegate to AI
- Judgment required → keep human

Automate incrementally.

---

## Summary

SIPO changes three things:

- **Correctness** — You can trace back to the authoritative source
- **Reproducibility** — Anyone gets the same result
- **Automation** — The boundary of what AI can handle is clear

Here is something you can do today:

Pick one process in your organization. Identify its Source.
"What is the original for this information?" — that question is the first step toward real process improvement.

**Only organizations that design their Sources can truly leverage AI.** That is my conclusion.
