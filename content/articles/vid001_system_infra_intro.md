## 1. インフラとは何か

「インフラ」はインフラストラクチャの略で、システムが動くための土台です。

道路や水道が社会インフラであるように、ITにおけるインフラは「アプリケーションが動くために必要な環境」を指します。具体的には、サーバ、ネットワーク、ストレージ、OS、ミドルウェアなどです。

多くの人が日常的に使っているメールやWebサービスも、このインフラの上で動いています。

## 2. サーバの基本

サーバとは、サービスを提供する側のコンピュータです。

### 物理形態による分類

- **タワー型**: オフィスに置ける箱型。小規模向け
- **ラック型**: データセンターのラックに積む。中〜大規模向け
- **ブレード型**: 薄型で高密度。大規模向け

### サーバの基本構造

どの形態でも中身は同じです：CPU（処理）、メモリ（一時記憶）、ストレージ（永続記憶）、NIC（ネットワーク接続）。そしてこれらを管理するOSが載っています。

## 3. 仮想化とコンテナ

### 仮想化とは

1台の物理サーバの上に、複数の仮想的なサーバを作る技術です。物理サーバを「マンション全体」とすると、仮想マシンは「各部屋」のようなものです。

メリットは、ハードウェアの利用効率が上がること。1台のサーバで複数のシステムを動かせます。

### コンテナとは

仮想マシンよりさらに軽量な仕組みです。OSを共有しながら、アプリケーションだけを独立させます。「部屋」ではなく「仕切り」に近いイメージです。

Dockerが代表的なコンテナ技術です。起動が速く、同じ環境をどこでも再現できるのが強みです。

## 4. クラウドの基本

クラウドとは、自分でサーバを持たず、インターネット経由でコンピューティングリソースを借りるサービスです。

### 3つのサービスモデル

| モデル | 提供範囲 | 例 |
|--------|----------|-----|
| **IaaS** | サーバ・ネットワークを提供 | AWS EC2, Azure VM |
| **PaaS** | 開発基盤まで提供 | Heroku, Google App Engine |
| **SaaS** | アプリケーションまで提供 | Gmail, Salesforce, Office 365 |

IaaSは「土地を借りて自分で家を建てる」、PaaSは「建売住宅を買う」、SaaSは「ホテルに泊まる」とイメージすると分かりやすいです。

## 5. APIとは

APIは Application Programming Interface の略です。

簡単に言えば「ソフトウェア同士が会話するための窓口」です。レストランの注文窓口のようなもので、メニュー（仕様）に従ってリクエストを出せば、結果が返ってきます。

例えば、天気予報アプリは気象庁のAPIからデータを取得しています。地図アプリはGoogle Maps APIを使っています。自分でデータを持つ必要がなく、必要な情報を必要なときに取得できます。

### REST API

現在最も広く使われているAPIの設計スタイルです。HTTPの仕組みを使い、URLでリソースを指定し、GET（取得）/ POST（作成）/ PUT（更新）/ DELETE（削除）で操作します。

## 6. Webの仕組み

Webは、ブラウザ（クライアント）とWebサーバの間で情報をやり取りする仕組みです。

### 基本的な流れ

1. ブラウザがURLを入力
2. DNSでIPアドレスを解決
3. Webサーバにリクエストを送信
4. サーバがHTMLを返す
5. ブラウザが画面を表示

### 3つのシステム形態

| 形態 | 特徴 | 例 |
|------|------|-----|
| **スタンドアロン型** | 1台で完結 | Excel, メモ帳 |
| **クライアント/サーバ型** | 専用ソフト＋サーバ | 会計ソフト, 基幹システム |
| **Web型** | ブラウザ＋Webサーバ | Gmail, Salesforce |

現在はWeb型が主流ですが、オフライン動作やレスポンス速度が求められる場面ではクライアント/サーバ型やスタンドアロン型にも利点があります。

## 7. アーキテクチャの基礎

アーキテクチャとは、システム全体の構造設計のことです。

建築でいう「設計図」にあたります。どの技術を使い、どう組み合わせ、どうスケールさせるか。これを決めるのがアーキテクチャです。

### よく使われるパターン

- **モノリシック**: 1つの塊として構築。シンプルだが大規模化すると辛い
- **マイクロサービス**: 機能ごとに独立したサービスに分割。スケーラブルだが複雑
- **サーバレス**: サーバ管理不要。イベント駆動で必要なときだけ動く

## まとめ

インフラは目に見えにくいですが、すべてのシステムの土台です。

この記事で押さえた7つの概念を理解しておけば、DXやシステム導入の会話で「何を言っているか分からない」という状態から脱却できます。

用語を「自分の言葉で説明できる」ことが、非エンジニアにとって最も大事なゴールです。
