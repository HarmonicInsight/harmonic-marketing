# AIで数字が出る＝正しい、ではない──AI会計自動化の落とし穴

tags: AI会計, 監査対応, 算出根拠, 業務コンサル, 建設業DX, 管理会計, 内部統制

---

## はじめに：「AIに聞いたら出ました」は通用しない

最近、GoogleスプレッドシートにGeminiが統合され、自然言語で数式を自動生成できるようになりました。
「A列とB列から合計を出して」と言えば、AIが勝手に数式を入れてくれる。

便利です。間違いなく便利です。

**でも、会計の世界では「数字が出る」と「数字が正しい」はまったく別の話です。**

監査法人に「この数字の根拠は？」と聞かれたとき、「AIが自動で計算しました」は回答になりません。
税務調査で「この経費の算出基準は？」と聞かれたとき、「スプレッドシートのAIに任せました」では通りません。

これは建設業に限らず、すべての業種に共通する問題です。
しかし、工事別原価管理が必要な建設業では、特にリスクが大きい。

---

## 1. 会計における「数字」の意味

会計で求められるのは、単に数字が存在することではありません。

| 要件 | 内容 |
|------|------|
| **算出根拠** | なぜその計算式なのか。どの基準に基づくのか |
| **計算手順** | どのデータを、どの順番で、どう処理したか |
| **承認フロー** | 誰が確認し、誰が承認したか |
| **変更履歴** | いつ、誰が、何を変更したか |
| **監査証跡** | 上記すべてが追跡可能であること |

AIが数式を自動生成すると、**算出根拠が消えます**。
なぜ「=A1*B1」なのか。なぜ「=SUMIF(...)」なのか。
AIは正しい数式を出すかもしれませんが、**なぜその数式なのかを説明してくれません**。

---

## 2. 監査で問われること

### J-SOX（内部統制報告制度）の観点

上場企業であれば、財務報告に関する内部統制の評価が義務づけられています。
ここで求められるのは：

- **統制活動の文書化** — 計算ロジックが明文化されていること
- **モニタリング** — 計算結果を誰がどう検証しているか
- **IT統制** — システムによる計算の正確性をどう担保しているか

AIが自動生成した数式は、この3つすべてに抵触するリスクがあります。

### 税務調査の観点

税務調査では、経費や売上の算出根拠を具体的に説明する必要があります。

「このAIツールが自動で計算しました」は、根拠の説明にはなりません。
**根拠とは、計算式の存在ではなく、計算式の妥当性の証明**です。

---

## 3. 建設業で特にリスクが高い理由

建設業の会計には、他業種にはない特殊性があります。

### ① 工事進行基準の計算

工事進行基準では、工事の進捗度に応じて売上を計上します。

```
当期売上 = 契約金額 × 工事進捗度
工事進捗度 = 発生原価 ÷ 見積総原価
```

この「見積総原価」は現場の判断に依存します。
AIが自動で数字を出したとしても、**その見積総原価の妥当性は人間が判断する**必要がある。

### ② 工事別原価管理

建設業では、工事ごとに原価を管理します。

- 材料費、外注費、労務費、経費の区分
- 共通仮設費の配賦基準
- 間接経費の配賦方法

これらの**配賦基準**を明確に定義しないまま、AIに「原価を計算して」と言っても、AIは何らかの数字を出します。
**出ますが、その配賦基準が正しいかどうかはわかりません。**

### ③ 完成工事原価報告書

税務申告に必要な完成工事原価報告書では、原価の内訳を正確に報告する必要があります。
ここで求められるのは、個々の原価項目の**積み上げ根拠**です。

AIが合計を出すことは簡単です。
でも、その合計の**内訳の正当性**を説明できなければ、報告書としての価値はありません。

## 4. AI自動化の「正しい使い方」

誤解のないように言いますが、AI自動化を否定しているわけではありません。
**順序の問題**です。

### やってはいけない順序

```
❌ AIツール導入 → 数字が出る → 「便利だ」 → 監査で指摘 → 根拠の後付け → 大幅な手戻り
```

### 正しい順序

```
✅ ① 算出基準の定義（会計基準・社内ルールに準拠）
   ② 計算手順の文書化（誰が見ても同じ結果になる手順書）
   ③ 承認フロー・監査証跡の設計
   ④ その上でAI自動化を適用
```

ステップ①〜③が整備されていれば、AIは強力な武器になります。
**定義された計算ロジックを、AIが正確に・高速に実行する。**
これが本来のAI活用です。

逆に、①〜③を飛ばしてAIを入れると、**後からの整備コストが導入コストの何倍にもなります**。
「大人の投資」が膨らむのは、この順序を間違えたときです。

---

## 5. 具体的に何をすべきか

### Step 1：算出基準書の作成

すべての主要な計算について、以下を明文化する：

- **計算式**（例：粗利 = 契約金額 - 実行予算原価合計）
- **準拠基準**（例：建設業会計基準 第○条、社内規程 第○号）
- **入力データの定義**（例：「契約金額」は変更契約を含む現契約金額合計）
- **例外処理**（例：追加工事発生時の取扱い）

### Step 2：計算手順書の整備

- データの取得元と取得方法
- 計算の実行手順（手動 or システム or AI）
- 検証方法（ダブルチェック、クロスチェック）
- 承認者と承認基準

### Step 3：AI活用ルールの策定

- AIが自動生成した数式は、必ず人間が検証してから適用する
- AIの計算結果は「下書き」として扱い、承認プロセスを経て「確定」とする
- AIが使用した計算ロジックをログとして記録する

---

## 6. まとめ：便利さの裏にあるリスクを見る

AI会計自動化の記事は、「便利になった」「関数知識が不要になった」というポジティブな面ばかりを強調します。

でも、業務コンサルの視点から見れば、**便利さの裏に監査リスクが潜んでいます**。

- 数字が出る ≠ 数字が正しい
- 計算できる ≠ 根拠がある
- 自動化された ≠ 統制されている

**AIは計算を速くする道具であって、算出根拠を作る道具ではありません。**

根拠なき自動化は、いずれ監査で破綻します。
そのとき発生するやり直しコストは、最初にきちんと整備するコストの比ではない。

DXと同じです。
**まず基準を整理する。手順を定義する。その上で自動化する。**
この順序を守れる会社だけが、AI自動化の恩恵を本当に受けられます。
