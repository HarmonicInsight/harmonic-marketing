# 努力が報われない人に足りない「やり直せる力」とは

タグ: 資料作成, 仕事術, 業務効率化, コンサル, スキルアップ

---

## 3時間かけた資料を、捨てられますか？

あなたは3時間かけて作った資料に「ここ、構成から間違ってるよ」と言われたら、どうしますか？

ほとんどの人は「修正で何とかしよう」とします。でも、デッサンが狂った絵は、どれだけ色を塗っても良い絵にはなりません。

資料も同じです。構成が間違っていたら、文言をいくら直しても「伝わる資料」にはなりません。

「でも3時間もかけたし……」

**この感覚こそが、努力が報われない最大の原因です。**

## 間違ったチャレンジには共通構造がある

こんな経験はありませんか？

- 新しいツールを導入 → マニュアルを読まずにいきなり使う → 「このツール使えない」
- 業務分析を任される → 手法を学ばずにいきなり着手 → 「時間が足りなかった」
- 転職・起業に挑戦 → 準備なしに飛び込む → 「自分には無理だった」

共通しているのは、**「準備」をすっ飛ばして、いきなり結果を出そうとしている**こと。

立場も能力も関係ありません。ジュニアコンサルタントも、経営者も、構造は全員同じです。

## 実証：Harmonic Insightの「パワポ虎の穴」

私たちHarmonic Insightのコンサルティングチームで、1ヶ月間の実験をしました。

**ルールはシンプル。毎日1枚、資料を作る。時間を計る。最短を追求する。**

ショートカット、コマンド配置、操作の型を全部自分の体で覚えていく。

結果はこうなりました。

- 1日目：15分
- 1週間後：5分（3倍速）
- 1ヶ月後：1〜2分

チーム全員、素直にやったら、すぐに効果が出ました。

### 本当に変わったのは「速さ」じゃない

一番大きな変化は、**資料を作ること自体が「苦」じゃなくなった**ことです。

1〜2分で作れるなら、間違っていても捨てられる。捨てられるから、常に最善のものが出せる。

つまり**「やり直しのコスト」がゼロに近づいた**のです。

**以前：** 3時間かけた → 直すのが怖い → 妥協して提出
**虎の穴の後：** 2分で作れる → 違ったら捨てる → 常にベストを出せる

## 「とりあえず病」はなぜ起きるのか

人間の思考の95%は無意識に起きていると言われています。脳は「動け」と命令する生き物です。準備＝動かない＝危険だと本能が判断してしまう。

その結果、こんな悪循環に陥ります。

1. 準備せずに始める
2. 非効率・失敗する
3. 時間を浪費する
4. 「準備する時間がない」
5. また準備せずに始める

これが**「とりあえず病」**です。あなたの日常にもありませんか？

- **転職：** 業界研究なしに応募 → まず自分の市場価値を把握すべき
- **英語：** いきなり英会話スクール → まず基礎文法と語彙を叩き込むべき
- **副業：** 「稼げるらしい」で開始 → まずビジネスモデルを理解すべき
- **資格：** 「とりあえず取る」 → まず「なぜこの資格か」を明確にすべき

## 正しいチャレンジの原理

正しい順番はこうです。

**基礎を徹底的に叩き込む → 作り直しコストがゼロに近づく → 失敗が怖くなくなる → 常にベストな結果が出せる**

これは資料作成だけの話ではありません。プログラミング、営業、マネジメント、データ分析、すべてに当てはまります。

## 正しいチャレンジの3条件

**1. 基礎を「体」に覚えさせる**
「知っている」と「できる」は別物です。考えなくても指が動くレベルまで叩き込む。

**2. 「計測」する**
計測しないと上達が見えません。上達が見えないと続きません。やったことには必ず時間を計ってください。

**3. 「捨てる練習」をする**
わざと作って、わざと捨てる。捨てる経験を重ねると、方向転換が怖くなくなります。

## あなたの「虎の穴」の作り方

**Step 1：「最小単位の成果物」を決める**
パワポなら1枚。プログラミングなら1つの関数。営業なら1通のメール。

**Step 2：「基礎動作」を洗い出す**
それを作るのに必要な操作・手順・知識を全部リスト化する。

**Step 3：「毎日1つ、計測付き」で回す**
毎日1つ作る。時間を計る。1ヶ月続ければ世界が変わる。

分野別の例を挙げると：

- **プログラミング：** 最小単位＝1つの関数 → 言語構文・IDE操作を叩き込む → 目標：5分→1分
- **営業・提案：** 最小単位＝1枚の提案書 → 顧客分析・構成パターンを覚える → 目標：30分→10分
- **マネジメント：** 最小単位＝1回の1on1 → 傾聴・質問技法を身につける → 目標：準備15分→5分
- **データ分析：** 最小単位＝1つのレポート → SQL・可視化・統計基礎を固める → 目標：2時間→30分

どの分野でも構造は同じです。

## やり直せる力は、地味で、退屈で、最強です

派手なチャレンジは映えます。でも、大抵うまくいきません。

地味に基礎を叩き込んで、計測して、捨てる練習をする。これをやった人だけが、**何度でもやり直せる人**になります。

あなたの「虎の穴」を、明日から始めてください。
